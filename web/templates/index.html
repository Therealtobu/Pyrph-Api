Protection Layers
Multiple layers of defense
🔀
PolyVM Bytecode
Custom VM with per-build randomized opcode mapping. Two builds of the same source produce completely different bytecode.
🔗
Nested VM
Stack + register hybrid VM inside the outer VM. Adds a second layer of indirection that defeats most automated analysis.
🔐
Native C Runtime
AES-128-CTR encrypted .so loaded via memfd_create. Whitebox key, LLVM passes, ptrace anti-debug. Never touches disk.
🧬
MBA Transforms
Mixed Boolean-Arithmetic replaces ops with cryptographically complex equivalents. Defeats Z3 and symbolic solvers.
🌀
Control Flow Flatten
Converts functions into random state machines with fake dead states. CFG unreadable to Ghidra/IDA.
🛡️
Anti-Debug
ptrace, pydevd, timing, stack depth, dis blocker, scattered sentinels. Multi-layer in both Python and C.
How It Works
Up and running in 4 steps
1

Install

Run pip install pyrph in Termux, terminal, or cmd.

2

Get your HWID

Run pyrph — it shows your machine ID. Copy it.

3

Get a key

Paste your HWID at pyrph.vercel.app/getkey for free, or contact admin for paid.

4

Activate & obfuscate

Run pyrph --activate PRF-XXXXX then pyrph again and enter your file.
